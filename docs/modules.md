## Module 1: Nginx (Gateway)

### Responsibility

- Public SNI stream routing on port 443 (layer 4, no TLS termination for Xray)
- TLS termination and HTTP virtual hosting for all *.ruyin.ai domains
- HTTP port 80: ACME challenge + redirect
- Reverse proxy to all internal services

### Config Structure

Nginx runs two listeners:

**Stream listener (port 443, in `nginx.conf` stream block):**
- `ssl_preread on` — reads SNI without terminating TLS
- Routes to `xray_backend` (port 10443) or `http_backend` (port 8443)

**HTTP listener (internal port 8443):**
- TLS termination with Let's Encrypt certs
- One `server {}` block per virtual host

### Virtual Host Configs

| Template | Domain | Upstream |
|----------|--------|---------|
| `ruyin-landing.conf.template` | `ruyin.ai` | static files in `nginx/html/ruyin-landing/` |
| `www-ruyin.conf.template` | `www.ruyin.ai` | static files in `nginx/html/www-ruyin/` |
| `vpn-portal.conf.template` | `vpn.ruyin.ai` | `umbra-portal:80` |
| `sub-marzban.conf.template` | `sub.ruyin.ai` | `umbra-marzban:8000` |
| `console.conf.template` | `console.ruyin.ai` | `umbra-marzban:8000` + 3-layer access control |
| `06-pass.conf.template` | `pass.ruyin.ai` | `umbra-vaultwarden:80` |
| `08-docs.conf.template` | `docs.ruyin.ai` | `umbra-docs:80` |

### Stream Config Spec

```nginx
stream {
    map $ssl_preread_server_name $backend {
        www.microsoft.com   xray_backend;
        default             http_backend;
    }

    upstream xray_backend {
        server umbra-xray:10443;
    }

    upstream http_backend {
        server 127.0.0.1:8443;
    }

    server {
        listen 443;
        proxy_pass $backend;
        ssl_preread on;
    }
}
```

### console.ruyin.ai — Three-Layer Access Control

Access to `console.ruyin.ai` requires passing all three layers in sequence:

```
Layer 1: VPN network restriction (Nginx allow/deny)
Layer 2: Nginx HTTP Basic Auth
Layer 3: Marzban web login
```

#### Layer 1: VPN Network Restriction

When a client connects via VLESS VPN and accesses `console.ruyin.ai`, the request is proxied through the Xray subprocess inside `umbra-marzban`. Nginx sees the source IP as a Docker internal address (`172.x.x.x` or `127.0.0.1`).

Direct access from any public IP is rejected at this layer.

```nginx
server {
    listen 8443 ssl;
    server_name console.ruyin.ai;

    ssl_certificate /etc/letsencrypt/live/console.ruyin.ai/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/console.ruyin.ai/privkey.pem;

    # Layer 1: Only allow Docker internal network (VPN-tunneled traffic)
    allow 172.16.0.0/12;
    allow 127.0.0.1;
    deny all;

    # Layer 2: Nginx Basic Auth
    auth_basic "Umbra Console";
    auth_basic_user_file /etc/nginx/.htpasswd_console;

    location / {
        # Layer 3: Marzban handles its own login
        proxy_pass http://umbra-marzban:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

#### Basic Auth Credential Generation

```bash
# Generate .htpasswd_console (run once, store result in DATA_DIR/private/)
htpasswd -c DATA_DIR/nginx/private/.htpasswd_console <admin-username>
chmod 600 DATA_DIR/nginx/private/.htpasswd_console
```

Mount into Nginx container:

```yaml
volumes:
  - DATA_DIR/nginx/private/.htpasswd_console:/etc/nginx/.htpasswd_console:ro
```

#### Access Flow Summary

```
Operator connects VPN → opens browser → console.ruyin.ai
  Layer 1: source IP = 172.x.x.x → PASS
  Layer 2: browser prompts Basic Auth credentials → PASS
  Layer 3: Marzban login form → PASS
  → Admin dashboard accessible
```

```
Anyone without VPN → console.ruyin.ai
  Layer 1: source IP = public IP → 403 DENIED
  → Never reaches Basic Auth or Marzban
```

### Docker Volumes

```yaml
volumes:
  - DATA_DIR/nginx/conf.d:/etc/nginx/conf.d:ro
  - DATA_DIR/nginx/stream.d:/etc/nginx/stream.d:ro
  - DATA_DIR/nginx/html:/usr/share/nginx/html:ro
  - DATA_DIR/letsencrypt:/etc/letsencrypt:ro
  - DATA_DIR/certbot:/var/www/certbot:ro
  - DATA_DIR/nginx/logs:/var/log/nginx
```

## Module 2: Xray-core

### Responsibility

- VLESS + REALITY + Vision proxy
- Multi-user, UUID-isolated clients
- Managed entirely by Marzban (Marzban writes `xray_config.json` and restarts Xray)

### Xray ↔ Marzban Relationship

Marzban manages Xray. The deployment flow is:

1. REALITY keys generated by `02-generate-reality.sh`
2. Keys written to `private/reality.json`
3. Marzban configured with REALITY keys at startup
4. Marzban renders its own `xray_config.json` and manages Xray process internally

**Marzban runs Xray as a subprocess** — do NOT run a separate `umbra-xray` container alongside Marzban. Marzban's container includes Xray.

### Xray Config (managed by Marzban)

Key inbound spec that Marzban must be configured with:

```json
{
  "tag": "VLESS_TCP_REALITY",
  "listen": "0.0.0.0",
  "port": 10443,
  "protocol": "vless",
  "settings": {
    "clients": [],
    "decryption": "none"
  },
  "streamSettings": {
    "network": "tcp",
    "security": "reality",
    "realitySettings": {
      "show": false,
      "dest": "www.microsoft.com:443",
      "xver": 0,
      "serverNames": ["www.microsoft.com"],
      "privateKey": "{{ REALITY_PRIVATE_KEY }}",
      "shortIds": ["{{ REALITY_SHORT_ID }}"]
    }
  },
  "sniffing": {
    "enabled": true,
    "destOverride": ["http", "tls", "quic"]
  }
}
```

### Port

| Port | Visibility | Usage |
|------|-----------|-------|
| 10443 | Internal only | VLESS+REALITY inbound |

### Marzban Container Volumes

```yaml
volumes:
  - DATA_DIR/marzban:/var/lib/marzban
```

## Module 3: Marzban

### Responsibility

- VPN user management (create, disable, delete, reset traffic)
- Subscription generation (Clash/Mihomo YAML, V2Ray JSON, etc.)
- Traffic accounting and expiry enforcement
- Xray config management (renders and hot-reloads Xray)
- REST API for automation
- Multi-node support (future: add more edge nodes)

### Image

```
gozargah/marzban:latest
```

### Key Environment Variables

```env
UVICORN_HOST=0.0.0.0
UVICORN_PORT=8000

SQLALCHEMY_DATABASE_URL=sqlite:////var/lib/marzban/db.sqlite3

XRAY_JSON=/var/lib/marzban/xray_config.json
XRAY_EXECUTABLE_PATH=/usr/local/bin/xray
XRAY_ASSETS_PATH=/usr/local/share/xray

SUDO_USERNAME={{ MARZBAN_ADMIN_USER }}
SUDO_PASSWORD={{ MARZBAN_ADMIN_PASSWORD }}

SUBSCRIPTION_URL_PREFIX=https://sub.ruyin.ai

CUSTOM_TEMPLATES_DIRECTORY=/var/lib/marzban/templates
CLASH_SUBSCRIPTION_TEMPLATE=clash/default.yml
```

### Marzban Subscription Template Location

```
DATA_DIR/marzban/templates/clash/default.yml
```

This file is rendered from `configs/marzban/clash-subscription.j2` — it contains the B++ proxy rules. See `design.md` for full rules content.

### Docker Config

```yaml
umbra-marzban:
  image: gozargah/marzban:latest
  restart: always
  env_file: .env
  volumes:
    - DATA_DIR/marzban:/var/lib/marzban
  networks:
    - umbra-net
```

## Module 4: Storage (SQLite)

### Decision

All services use SQLite on this 1C1G node. No PostgreSQL container.

| File | Service |
|------|---------|
| `DATA_DIR/marzban/db.sqlite3` | Marzban |
| `DATA_DIR/vaultwarden/data/db.sqlite3` | Vaultwarden |

### Backup

SQLite databases are backed up via file copy in `07-backup.sh`. Vaultwarden's full `data/` directory is archived (includes attachments and sends).

## Module 8: Certbot

### Responsibility

- Issue Let's Encrypt certs for all 9 domains
- Auto-renew via cron

### Issue Command

All domains in a single cert (or separate certs per domain — separate recommended for easier rotation):

```bash
# Issue per domain (recommended)
certbot certonly --webroot \
  --webroot-path DATA_DIR/certbot \
  --email "$CERTBOT_EMAIL" \
  --agree-tos --non-interactive \
  -d vpn.ruyin.ai

# Repeat for each domain
```

Or as a wildcard cert (requires DNS challenge, more complex):

```bash
certbot certonly --dns-cloudflare \
  --dns-cloudflare-credentials /path/to/cf-creds.ini \
  -d "*.ruyin.ai" -d "ruyin.ai"
```

**Recommendation:** Per-domain webroot for simplicity in v1.0. Consider wildcard in v1.1 if managing 9 certs becomes maintenance burden.

### Renewal Script

```bash
certbot renew --quiet
docker exec umbra-nginx nginx -s reload
```

### Cron

```cron
17 3 * * * /srv/vxture/repo/umbra/scripts/renew-cert.sh >> /var/log/umbra-cert-renew.log 2>&1
```
